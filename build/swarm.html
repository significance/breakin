Ethereum Swarm is a decentralized storage and distribution network.

Think of it as BitTorrent meets a global hard drive, but with economic incentives built in. Nodes get paid to store and serve data. No single server, no single point of failure, no kill switch.

Your files get split into 4KB chunks, content-addressed by their cryptographic hash, and scattered across a peer-to-peer network of Bee nodes running across the planet. Tamper with a single byte and the hash breaks -- data integrity is mathematically guaranteed.

KEY INSIGHT: Once data is uploaded to the Swarm, the original uploader can disappear. The network continues to serve the content autonomously. "Upload and disappear" is a feature, not a bug.

TERM MEANING
Bee The official Swarm node client. You run one to join the network. Written in Go. Light nodes can upload/download without storing data for others.
BZZ The native ERC-20 token of Swarm. Used to buy postage stamps and for all economic activity on the network. Also the protocol prefix: bzz://
Chunk The fundamental unit of storage. Every file is split into 4KB chunks, each content-addressed by its hash. Everything is chunks. Chunks all the way down.
Postage Stamp Prepaid storage credits. You buy a batch of stamps with BZZ tokens to upload data. Like buying actual postage before mailing a letter. Choose amount + TTL (time to live).
Feed A mutable pointer to immutable content. Lets you update your website at a fixed address without changing the URL. Like DNS but decentralized.
Manifest A data structure mapping URL paths to content hashes. Acts as a decentralized webserver config -- routes /index.html to the right chunks.
DISC Distributed Immutable Store for Chunks. The protocol that retrieves chunks from the nearest Bee nodes. Chunks migrate toward nodes whose address is closest to the chunk hash.
Gnosis Chain The L1 blockchain where Swarm's incentive layer runs. Transaction fees paid in xDAI (stablecoin). Fast and cheap.
Gateway An HTTP bridge to the Swarm network. Lets normal browsers access Swarm content via regular HTTPS URLs without running a Bee node locally.
FUN FACT: The Swarm team really committed to the bee theme. The node is called Bee. The token is BZZ. The test tokens are gBZZ. The protocol is bzz://. The testnet was called "Queen". They really like bees.
